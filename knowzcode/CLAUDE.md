# KnowzCode вЂ” Framework Source

## What This Repository Is

This is the **source repository** for the KnowzCode framework вЂ” a platform-agnostic AI development methodology installed via plugin (`/plugin install knowzcode@knowz-marketplace`) or script (`npx knowzcode`).

Working here means modifying the template framework itself, not using it on a project. End users get their own CLAUDE.md generated by `/knowzcode:init` from `knowzcode/platform_adapters.md`.

For product overview, see [README.md](README.md). For detailed docs, see `docs/`.

## Key Development Conventions

- **`knowzcode/`** contains template files copied into user projects on `/knowzcode:init` вЂ” changes here affect all new installations
- **`skills/`** contains skill definitions вЂ” each `<name>/SKILL.md` defines one skill (8 user-invocable + 2 trigger). For MCP vault features, install the knowz plugin: `claude plugin install knowz`
- **`agents/`** contains agent definitions вЂ” each `.md` file defines one specialized agent
- **`knowzcode/platform_adapters.md`** holds the adapter templates for all 6 platforms (Claude Code, Gemini, Codex, Cursor, Copilot, Windsurf)
- **`knowzcode/prompts/`** holds phase prompt templates used by commands and agents
- **`knowzcode/knowzcode_loop.md`** is the core methodology вЂ” must stay platform-neutral (no Task tool, no subagent references, no Claude Code-specific syntax)

## Agent Teams (Teammate Instructions)

**If you were spawned as a teammate:**
1. Your spawn prompt identifies your role and agent definition file вЂ” read it first
2. Follow the conventions in your spawn prompt (WorkGroup updates, prefix rules, completion reporting)
3. Follow the instructions in your assigned task
4. Report results by updating the WorkGroup file and completing your task with a summary
5. For detailed team conventions, see `knowzcode/claude_code_execution.md`
